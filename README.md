# AI Prompt Engineering using RTCFR Framework

## Overview

This repository demonstrates **structured prompt engineering** using the **RTCFR framework**, a simple yet powerful approach to designing high-quality prompts for large language models (LLMs).

RTCFR helps transform vague prompts into **clear, structured instructions** that produce more reliable and context-aware AI responses.

This repository includes practical examples showing how RTCFR can be applied to **real-world scenarios**, such as generating a pediatric nutrition feeding plan.

---

# What is RTCFR?

RTCFR is a **prompt structuring method** that improves AI outputs by clearly defining the problem context and expected response.

RTCFR stands for:

| Component | Meaning | Purpose |
|----------|---------|---------|
| **R — Role** | Define who the AI should act as | Provides expertise and perspective |
| **T — Task** | Define what the AI must do | Clarifies the objective |
| **C — Context** | Provide background information | Helps AI understand constraints and environment |
| **F — Few-Shot Examples** | Show examples of expected inputs or outputs | Guides response structure |
| **R — Report / Tone** | Define the output format and communication style | Ensures consistent and useful responses |

Using RTCFR ensures prompts are **structured, precise, and repeatable**, which significantly improves response quality.

---

# Why RTCFR Matters

Most prompts fail because they are **too vague**.

Example of a weak prompt:
 Give me a feeding plan for a baby

 
The AI lacks:

- context
- constraints
- format expectations
- expertise definition

Using RTCFR:
ROLE: Pediatric nutritionist
TASK: Create a feeding schedule
CONTEXT: 9-month baby transitioning to solids
FEW SHOTS: Example feeding structure
REPORT: Provide a 7-day table with nutrition details

The result becomes **far more structured and useful**.
